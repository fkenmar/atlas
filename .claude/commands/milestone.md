Audit the current milestone:

1. Read STATUS.md for the current milestone, then docs/PRD.md §9 for that milestone's exit criteria.
2. Report each criterion as ✅ or ❌ **with concrete evidence**: file paths, passing test names, benchmark numbers, command output. No vibes — a criterion you can't evidence is ❌.
3. List the shortest path to completing the ❌ items: concrete tasks, ordered by dependency, sized for the ~10 hrs/week budget.
4. If STATUS.md's checkboxes disagree with the evidence, flag the mismatch and propose the correction (don't silently edit).
